# 変更履歴

このファイルは [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) の形式に概ね従い、バージョン番号は [セマンティックバージョニング](https://semver.org/lang/ja/) を採用する。新しい版を上に積む。

## [0.2.0] — 現行

ローカル MVP に、会話の記憶・キャラクター性・外部情報の取り込みを加えた版。つくよみという人格を中心に、過去の会話を圧縮して覚え、ユーザーとの関係を数値で持ち、天気などを会話に織り込む。実際の音声出力には引き続き GPT-SoVITS の参照音声と各ローカルサービスが要る。

### 追加

- 会話記憶の3層構成。直近はそのまま文脈に残し、押し出された分はローカルの軽量 LLM が箇条書きへ圧縮して短期記憶へ、短期記憶が一定量たまると Gemini が長期記憶へ統合する。不変記憶（人格・口調）＞長期記憶＞短期記憶の優先度でプロンプトを組み立て、`data/memory.json` に永続する。
- 関係性パラメータ。親密度・友情・信頼・敬意（0〜100）と、その日の気分（-50〜50）を持つ。毎ターン背景で軽量 LLM が発話と状況から小さく調整し、現在値に応じて距離感と口調を変える。親密度が閾値以上のときは、より親密な表現を許容する。`data/relationship.json` に永続する。
- API 検索の基盤。発話を意図判定し、天気の質問なら OpenWeather の現在天気を取得して、そのターン限定の文脈として注入する。プロバイダを足して拡張できる。
- 時刻と日付の認識。現在時刻をプロンプトへ与え、時間に合わない挨拶には戸惑って理由を尋ねる。応答中の `2026-06-28` のような形式は、今日・明日・明後日・明々後日、それ以降は日付のみ、月をまたぐ場合は月と日へ会話的に整える。
- 日本語特化の LLM 選択肢。NVIDIA Nemotron Nano 9B v2 (Japanese) を Ollama 経由でフロント LLM として使用できる。推論トレースを出す reasoning モデル向けに、`<think>` 区間を除去して読み上げや履歴へ流さない。
- 起動時ウォームアップ。LLM・STT・TTS・VAD の初回コールド負荷を会話開始前に消化し、最初の応答の遅延を抑える。各段の所要時間を計測するログも加えた。
- `.env` からの設定読み込み（`GEMINI_API_KEY`、`OPENWEATHER_API_KEY`）。固有名詞の読み・区切りを安定させる TTS 読み置換。

### 変更

- 人格を「ことは」から「つくよみ」へ改名し、丁寧で穏やかな癒し系の口調に整えた。
- 記憶の圧縮と関係性の分析には軽量モデル（`qwen3.5:4b`）を用い、フロントの応答モデルと分離した。

### 修正

- 音声認識が `(segments, info)` のタプルを返していたため、文字列へ整形するようにした。
- 時刻のト書き（例「（02:15 ごろ）」）が読み上げられていたのを抑止した。

### 検証

- Python のユニットテストが 149 件成功する。

## [0.1.0] — 2026-06-26

ローカルのみで完結するリアルタイム音声対話の MVP と、デスクトップ・オーバーレイの基盤を整備した初期開発版。実際に音声を出力するには GPT-SoVITS の参照音声と各ローカルサービスの起動が必要であり、Discord 対応はこの先に予定する。

### 追加

- リアルタイム音声対話パイプライン。Silero VAD による発話区間検出、faster-whisper による音声認識、Ollama の Qwen3.5 によるストリーミング応答、文分割、GPT-SoVITS による音声合成までを通す。
- オーケストレータ。1ターンを文単位の3段パイプライン（文分割、合成、再生）で処理し、再生中の合成を重ねて発話開始までの遅延を抑える。STT 失敗は沈黙としてスキップ、本文の合成失敗はフォールバック発話、各段にタイムアウトを設ける。
- barge-in。再生中も VAD を動作させ、継続発話を検知したら生成と再生を停止してキューを破棄し、割り込みの冒頭を次の認識へ引き継ぐ。VAD ストリームは利用者別・用途別に分離し、境界でリセットする。
- ローカル入出力。sounddevice によるマイク入力とスピーカー再生、起動口 `local_app`、起動時の疎通チェック `health`、環境診断 `diagnostics`。
- デスクトップ・オーバーレイの Python 連携 (SP2)。aiohttp の WebSocket サーバ `OverlayBridge` が状態と口パク振幅を配信する。声ループ側は既定で no-op のため、オーバーレイ無効時の挙動は変わらない。
- デスクトップ・オーバーレイの描画基盤 (SP1)。Electron と three.js、@pixiv/three-vrm により、透明・最前面・クリックスルーの窓で VRM を表示し、SP2 の WebSocket を購読して状態と口パクを反映する。Python なしでも単体で起動できる。
- デスクトップ・オーバーレイのデスクトップ干渉 (SP3)。キャラをドラッグで移動でき、タスクバーの上を自律的に歩き、手を離すと重力で落下して着地する。投げると初速がのる。キャラの上だけ操作を受け、それ以外はクリックが背面へ素通しする。
- ドキュメント。設計書と実装計画、日英の README、Apache-2.0 ライセンス。

### 備考

- フロント LLM の既定は `qwen3.5:4b`、音声認識は faster-whisper の `large-v3-turbo`、GPU は RTX 4080 16GB を想定する。
- ユニットテストは Python 側が 84 件、オーバーレイ側が vitest で 19 件成功する。
